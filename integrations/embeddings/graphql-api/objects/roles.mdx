---
title: "Roles"
---

Roles control what a user can see and do within a WisdomAI workspace. When provisioning embedded users, you pass a role ID in `roleAssignments` to grant the appropriate level of access.

## Role reference

| Role | ID | Scope | Description |
|------|----|-------|-------------|
| `SITE_ADMIN` | `00000000-0000-0000-0000-000000000014` | Global | Full access across all organizations. Can create and delete workspaces. |
| `ADMIN` | `00000000-0000-0000-0000-000000000001` | Organization | Full org-level access. Cannot create or delete workspaces. |
| `DATA_ADMIN` | `00000000-0000-0000-0000-000000000004` | Domain | Full domain access. No connection write or delete, no IAM write or delete. |
| `CONNECTION_ADMIN` | `00000000-0000-0000-0000-000000000005` | Connection | Manage a specific connection (read, write, delete, refresh). Cannot create connections or list others. |
| `CREATOR` | `00000000-0000-0000-0000-000000000006` | Any | Create domains and connections. No query or dashboard edit access. |
| `EXPLORER` | `00000000-0000-0000-0000-000000000002` | Domain | Query a domain with SQL editing and AI workstream access. |
| `BASIC_EXPLORER` | `00000000-0000-0000-0000-000000000015` | Domain | Query a domain. No SQL editing, no AI workstream. |
| `VIEWER` | `00000000-0000-0000-0000-000000000003` | Organization | Read-only across agents, chats, dashboards, schedules, and evaluations. Cannot create or modify anything. |
| `OBSERVER` | `00000000-0000-0000-0000-000000000007` | Organization | Read-only access to workspace, IAM, and data tables. Cannot view dashboards or chat history. |
| `STORY_VIEWER` | `00000000-0000-0000-0000-000000000008` | Dashboard | View a specific dashboard. |
| `STORY_WRITER` | `00000000-0000-0000-0000-000000000009` | Dashboard | Edit a specific dashboard (read, write, clone). Cannot create new dashboards. |
| `SCHEDULE_VIEWER` | `00000000-0000-0000-0000-000000000010` | Schedule | View a specific schedule. |
| `SCHEDULE_WRITER` | `00000000-0000-0000-0000-000000000011` | Schedule | Edit a specific schedule. Cannot create new schedules. |
| `AGENT_VIEWER` | `00000000-0000-0000-0000-000000000012` | Agent | View a specific agent. |
| `AGENT_WRITER` | `00000000-0000-0000-0000-000000000013` | Agent | Edit a specific agent. |

<Note>
  Custom roles are not yet supported.
</Note>

## Used by

Role IDs are passed in the `roleAssignments` field of the following mutations:

- [`createUsers`](/integrations/embeddings/graphql-api/mutations/user/create-users) тАФ assign a role when provisioning a new embedded user
- [`setUserAttributes`](/integrations/embeddings/graphql-api/mutations/user/set-user-attributes) тАФ update the role of an existing embedded user
- [`addScopeRoleAssignmentsForSharing`](/integrations/embeddings/graphql-api/mutations/dashboard/add-scope-role-assignments-for-sharing) тАФ grant a scoped role on a specific dashboard
- [`removeScopeRoleAssignmentsForSharing`](/integrations/embeddings/graphql-api/mutations/dashboard/remove-scope-role-assignments-for-sharing) тАФ remove a scoped role from a specific dashboard
