---
title: "Add Table to Domain"
description: "Add tables from data source connections to existing domains for analysis"
---

Add a specific table from a data source connection to an existing domain, making it available for analysis and questioning within that domain's context.

## Overview

The `addTableToDomain` mutation links a table from your data warehouse to a domain. This enables the table's data to be included when asking questions or performing analysis within that domain.

## Mutation

```graphql
addTableToDomain(
  domainId: ID!
  connectionId: ID!
  databaseName: String!
  schemaName: String!
  tableName: String!
): AddTableToDomainPayload!
```

### Arguments

<ParamField path="domainId" type="ID!" required>
  The unique identifier of the domain to add the table to
</ParamField>

<ParamField path="connectionId" type="ID!" required>
  The unique identifier of the connection containing the table
</ParamField>

<ParamField path="databaseName" type="String!" required>
  The name of the database containing the table
</ParamField>

<ParamField path="schemaName" type="String!" required>
  The name of the schema containing the table
</ParamField>

<ParamField path="tableName" type="String!" required>
  The name of the table to add to the domain
</ParamField>

### Response

Returns an `AddTableToDomainPayload` with success status and error information:

```graphql
type AddTableToDomainPayload {
  userErrors: [UserError!]!
  success: Boolean!
}
```

## Complete Domain Setup Workflow

```graphql
# 1. Refresh connection to discover new tables
mutation {
  connectionRefresh(connectionId: "conn_abc123") {
    success
  }
}

# 2. Check refresh completion
query {
  connectionLastRefreshStatus(connectionId: "conn_abc123") {
    refreshStatus
  }
}

# 3. Create domain
mutation {
  createDomain(name: "Sales Analytics")
}

# 4. Add multiple tables to domain
```
