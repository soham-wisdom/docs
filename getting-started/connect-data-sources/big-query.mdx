title: "Connect to Google BigQuery"
---

WisdomAI connects to Google BigQuery via a Google Cloud Platform (GCP) Service Account to perform background system operations such as data crawling and metadata discovery. To execute end-user queries, administrators can either continue using the shared service account or enable Google OAuth to authenticate individual users with their own Google accounts.

## Prerequisites

Before connecting WisdomAl to BigQuery, make sure to have:

- An active Google Cloud Project.
- The BigQuery API enabled within your Google Cloud Project.

Find below the steps to activate and configure a Google Cloud Platform (GCP) Service Account and generate the JSON key:

<AccordionGroup>
  <Accordion title="Create a GCP service account.">
    A service account is a special type of Google account that an application, like WisdomAl, can use to make authorized API calls. To do this:

    1. Navigate to the GCP Console.
    2. Go to **IAM & Admin** \> **Service Accounts**.
    3. Click **\+ CREATE SERVICE ACCOUNT**.
    4. **Service Account Name:** Choose a descriptive name, such as "Wisdom-bigquery".
    5. **Service Account ID:** This will be auto-generated based on the name.
    6. **Service Account Description:** (Optional) Add a description, e.g., "Service account for WisdomAl BigQuery connectivity."
    7. Click **CREATE AND CONTINUE**.
  </Accordion>

  <Accordion title="Assign IAM roles to the service account.">
    To grant WisdomAl the necessary permissions to read data and run queries, assign the following roles to the service account created in the previous step. It's recommended to assign these roles at the project level for broad access, or you can restrict access to specific datasets if needed.

    - **BigQuery Data Viewer:** This role grants permission to view BigQuery data and metadata.
    - **BigQuery Job User:** This role allows the service account to run BigQuery jobs, including queries.
    - **BigQuery Read Session User:** This role is crucial for efficient data retrieval using the BigQuery Storage API.

    Follow these steps to Assign Roles (at Project Level):

    1. After creating the service account, on the "Grant this service account access to project" step, select the following roles from the **Select a role** dropdown:
       - **BigQuery Data Viewer**
       - **BigQuery Job User**
       - **BigQuery Read Session User**
    2. Click **DONE**.

    ### Restrict access to specific datasets only

    If you want WisdomAl to only access certain datasets within your project, you can skip assigning the BigQuery roles at the project level and instead grant access directly to the specific datasets:

    1. Navigate to BigQuery in the GCP Console.
    2. Select the desired dataset.
    3. Click **SHARE DATASET**.
    4. In the **Add principals** field, enter the email address of the service account you created.
    5. Assign the **BigQuery Data Viewer** and **BigQuery Read Session User** roles to the service account for this specific dataset.
    6. Assign **BigQuery Job User** at the project level.
    7. Repeat for all datasets WisdomAl needs to access.
  </Accordion>

  <Accordion title="Generate a JSON key for the service account.">
    WisdomAl will use a JSON key file for authenticating with BigQuery.

    1. Navigate back to **IAM & Admin** \> **Service Accounts**.
    2. Click on the service account you created (e.g., Wisdom-bigquery).
    3. Go to the **KEYS** tab.
    4. Click **ADD KEY \> Create new key**.
    5. Select **JSON** as the key type.
    6. Click **CREATE**.
    7. A JSON key file will be downloaded to your computer.
       <Warning>
         This file contains sensitive credentials.
       </Warning>
  </Accordion>
</AccordionGroup>

## Connect WisdomAI to BigQuery

BigQuery OAuth allows individual WisdomAI users to authenticate with their own Google account when running queries, rather than relying on a shared service account. This provides granular access control and a per-user audit trail.

Once you have set up the prerequisites, you can configure the connection in WisdomAI:

1. In WisdomAI, navigate to the **Connections** section and click **Add Connection**.
2. Add the Connection name and select **BigQuery** as the data source type.
3. Fill in the connection details in the "BigQuery connection details" section:
   - **Connection Name:** Choose a descriptive name for your connection (e.g., "BigQuery Prod").
   - **Project Id:** Provide your Google Cloud Project ID (this is usually found on your GCP console dashboard).
   - **Service Account Info JSON:** Paste the contents of the JSON key file.
   - **Dataset Filters (Optional):** Specify any Dataset Filters to further restrict the datasets WisdomAl crawls.
4. Click **Save and Sync metadata**. WisdomAl will use the provided credentials to connect to BigQuery, scan the metadata of the specified datasets and tables.

## Connect with BigQuery OAuth

You can use BigQuery OAuth to authenticate with your own Google account when running queries, instead of relying on a shared service account. This gives you more granular access control and a clearer per-user audit trail.

<Note>
  A service account JSON is still required for WisdomAI crawl jobs and metadata discovery. Complete the **Prerequisites** section above before proceeding.
</Note>

Enabling OAuth requires you to work across two systems: WisdomAI and the Google Cloud Console. The steps below guide you through both.

To start the OAuth connection setup in WisdomAI, follow these steps:

1. In WisdomAI, navigate to **Connections** and click **Add Connection**.
2. Enter a **Connection name** and select **BigQuery** as the data source.
3. Paste your **Service Account Info JSON** in the corresponding field.
4. Select **Enable OAuth for user queries (optional)**. The **Google OAuth Client ID** and **Google OAuth Client Secret** fields appear. You will populate these after creating an OAuth client in Google Cloud.

<Frame>
  <img
    src="/images/add-connection-oauth.png"
    alt="Enable OAuth for user queries checkbox"
    lightAlt="Enable OAuth for user queries checkbox"
    darkAlt="Enable OAuth for user queries checkbox"
  />
</Frame>

Create an OAuth 2.0 client in Google Cloud:

1. Go to the Google **Auth Platform** in the Google Cloud Console.
2. Navigate to **Clients** and click **Create client**.
3. Set **Application type** to **Web Application**.
4. Enter a name for the client, for example `WisdomAI`.
5. Leave **Authorized JavaScript origins** empty.
6. Under **Authorized redirect URIs**, add the following URI:
   ```text
   https://api.descope.com/v1/outbound/oauth/callback
   ```
7. Click **Create**.

<Frame>
  ![google cloud console oauth client creation](/images/create-client-oauth.png)
</Frame>

After the OAuth client is created, Google Cloud displays a confirmation dialog with your credentials.

1. Copy the **Client ID**.
2. Copy the **Client Secret**.
3. Optionally, click **Download JSON** to save the credentials locally for reference.

<Frame>
  ![Oauth Client Created](/images/oauth-client-created.png)
</Frame>

After that, enter credentials and authenticate in WisdomAI:

1. Return to the BigQuery connection form in WisdomAI.
2. Paste the **Client ID** and **Client Secret** into the corresponding fields.
3. Click **Connect with Google**.

<Frame>
  ![Client Id Secret](/images/client-id-secret.png)
</Frame>

4. In the Google Sign-In popup that appears, select the Google account you want to use for authentication.

<Frame>
  <img

</Frame>

To save the connection, follow these steps:

1. After successful Google authentication, you are redirected back to WisdomAI.
2. Verify that the OAuth connection status shows as connected.
3. Click **Save** to complete the connection setup.

## Security considerations and best practices

- **Least Privilege:** Always follow the principle of least privilege. Only grant the necessary roles and permissions to the service account. If dataset-level access is sufficient, avoid granting project-level roles.
- **Secure Key Management:** The JSON key file is highly sensitive. Treat it like a password. Avoid sharing it via insecure channels like unencrypted email. Use secure methods like LastPass or other secure file sharing services.

## Troubleshooting common issues

Facing issues? Here are solutions to some frequently encountered problems.

<AccordionGroup>
  <Accordion title="Authentication failed or invalid credentials.">
    If you're facing this issue, try:

    - Ensure the JSON key file you pasted is complete and correct. No extra characters, spaces, or missing lines.
    - Verify that the `project_id` in the JSON key matches the project ID you're trying to connect to.
  </Accordion>

  <Accordion title="Permission denied errors">
    If you're facing this issue, try:

    - Double-check that the service account has all three required roles (**BigQuery Data Viewer**, **BigQuery Job User**, **BigQuery Read Session User**) assigned correctly at either the project or dataset level.
    - Confirm that the BigQuery API is enabled in your Google Cloud Project.
  </Accordion>

  <Accordion title="Dataset not found or table not found">
    If you're facing this issue, try:

    - Verify the spelling of the dataset or table names.
    - If using dataset filters in WisdomAl, ensure the dataset is included in the filter.
    - Confirm that the service account has permissions on the specific dataset you are trying to access.
  </Accordion>
</AccordionGroup>
