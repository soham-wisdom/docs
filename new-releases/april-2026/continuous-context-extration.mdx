title: "Continuous Context Extraction"
---

Domain knowledge doesn't stay accurate on its own. Metric definitions, business rules, and table descriptions live across GitHub repositories, dbt YAML files, Looker models, and data catalogs, the moment any of those sources changes, your domain starts falling behind.

Continuous Context Extraction (ACE) closes that gap. You can now connect your domain directly to the systems where your business logic lives. ACE runs extraction on a scheduled or on-demand basis, detects what changed since the last run, and surfaces only new signals for your review. This means your domain stays perpetually aligned with the source of truth without requiring you to manually export or upload a file.

## How it works

Continuous Context Extraction follows four stages from connection to a published domain update.

<Steps>
  <Step title="Connect a source" icon="plug">
    On your domain's **Context Extractor** page, select the **Connections** tab and click **Add Connection**. A five-step wizard walks you through choosing a source type, authenticating (API key, OAuth, or Personal Access Token), scoping the extraction (a specific repository or org-wide), setting a schedule, and confirming. Credentials are validated immediately and the connection appears with **PENDING** status.
  </Step>
  <Step title="Run extraction" icon="arrows-rotate">
    Once connected, ACE runs extraction automatically on your configured schedule. The first run is a full scan of the connected source. Subsequent incremental runs process only changes since the last extraction staing lightweight and focused. You can trigger a manual extraction at any time using **Extract Now**. The connection status updates in real time: **EXTRACTING** while the job runs, **AWAITING REVIEW** when it completes.
  </Step>
  <Step title="Review conflicts and duplicates" icon="list-check">
    When extraction completes, ACE compares every incoming context object against your existing domain. Conflicting entries surface as update recommendations, suggested changes for your review before anything is applied. After conflict resolution, an embedding-based similarity pass flags semantically equivalent entries as duplicate candidates. You resolve each item in the review queue: accept, decline, merge, keep both, or defer. No changes are published until you explicitly approve.
  </Step>
  <Step title="Publish and version your domain" icon="rocket">
    After clearing the review queue, click **Publish**. ACE increments your domain version number, stores a full snapshot, and updates chat and dashboards immediately. Every published version is retained in the **Version History** panel. If a published version introduces issues, select any prior version and restore it in a single confirmation step. Rollback is append-only and nothing is permanently deleted.
  </Step>
</Steps>

<Note>
  Nothing in your domain is ever updated automatically. Every extraction passes through a human-in-the-loop review. Conflicts must be resolved and the dedup queue cleared before a new version is published. There is no auto-merge in the current release.
</Note>

## Key capabilities

• **Source connections**: Connect to GitHub using a read-only Personal Access Token (PAT), scoped to a single repository or an entire organization. Looker, dbt, Atlan, Glean, and Collibra are rolling out in the same sprint cycle and will appear in the connection wizard as they go live. No write access is required.

• **Scheduled and on-demand extraction**: Extraction runs daily at 2:00 AM local time by default; adjust the time during connection wizard setup. **Extract Now** triggers an immediate incremental run at any time, independent of the schedule. The first run is a full scan of the connected source; subsequent runs process only changes since the last extraction. Jobs run as persistent background processes; you can close your browser mid-extraction and return later, as status is saved every 60 seconds.

• **Conflict resolution**: After each extraction, ACE compares incoming context objects against existing domain entries by name. Definition mismatches are flagged as conflicts and surfaced as update recommendations in the review queue. The domain is not updated until every conflict is resolved or deferred. All resolution decisions are logged with user ID and timestamp. A dedicated side-by-side diff view with options to accept either source, write a merged definition, or defer is on the near-term roadmap.

• **Semantic duplicate detection**: After conflict resolution, ACE runs an embedding-based similarity pass across all context objects in the domain. Entries that mean the same thing, even if worded differently across sources, are flagged as duplicate candidates. Each candidate pair shows a similarity score and a suggested canonical definition; you can accept a merge, keep both entries, or defer. The dedup queue must be cleared before publishing a new domain version. No merges happen automatically.

• **Version history and rollback**: Each published extraction increments the domain version number and stores a full snapshot. The **Version History** panel shows version number, publish timestamp, author, change count, and a one-line summary; diffs between any two versions are available on demand. To roll back, select a prior version and confirm in a single step; rollback creates a new version entry rather than overwriting history. History is append-only; nothing is permanently deleted. Chat and dashboards reflect the rolled-back domain immediatel.

## Target users

Continuous Context Extraction is designed for:

• **Domain Admins** who manage the knowledge layer for their WisdomAI domain and need it to stay synchronized with evolving business logic across engineering and analytics sources.

• **Data and analytics teams** that maintain metric definitions, JOIN logic, and business rules in GitHub, dbt, or Looker and want domain updates to flow automatically from source changes rather than manual uploads.

Explorers and analysts are not required to take any action, they benefit through improved answer quality and richer **Context used** explanations on query responses.

## FAQs

<AccordionGroup>
  <Accordion title="What sources are supported today?">
    GitHub is live in the current release. Looker and dbt are available via the GitHub connector since they are typically stored in customer Git environments. Atlan, Glean, and Collibra are shipping in the same sprint cycle and will appear in the connection wizard as they go live.
  </Accordion>

  <Accordion title="Does anything get published to my domain automatically?">
    No. Every extraction goes through a human-in-the-loop review. Conflicts must be resolved and the dedup queue cleared before any version is published.
  </Accordion>

  <Accordion title="What happens if I close my browser mid-extraction?">
    Nothing is interrupted. Extraction jobs run as persistent background processes. Status is persisted every 60 seconds. You can close your browser, come back later, and the job will still be running or waiting for your review when you return.
  </Accordion>

  <Accordion title="What is the difference between a conflict and a duplicate?">
    A conflict is when the same named term has different definitions across sources, it's  detected by name match. A duplicate is when two differently-named entries are semantically equivalent it's detected by embedding similarity. Both require your review before a new version is published.
  </Accordion>

  <Accordion title="What if I publish something and later realize it was wrong?">
    Use the rollback feature. Open the **Version History** panel, find the prior version you want, and restore it with a single confirmation step. Rollback creates a new version entry. History is append-only, so nothing is ever permanently deleted.
  </Accordion>

  <Accordion title="Can I have multiple sources connected at once?">
    Yes. You can have multiple active connections simultaneously. For example, GitHub and Looker at the same time. Each runs its own extraction job. All extracted suggestions enter the same conflict and dedup review queue, which spans across sources.
  </Accordion>

  <Accordion title="What permissions does ACE need for a GitHub connection?">
    A Personal Access Token with read access to the repositories you want to extract from. For org-wide extraction, a token with `repo` scope on the organization. No write permissions are required at any point.
  </Accordion>

  <Accordion title="How is this different from the file upload Context Extractor?">
    The original Context Extractor is a one-time, file-based flow: export a file, upload it, review suggestions, and publish. Continuous Context Extraction connects ACE directly to the live source system, extracts automatically on a schedule, and keeps the domain continuously aligned, no manual export or file handling required.
  </Accordion>
</AccordionGroup>
