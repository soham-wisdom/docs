title: "Continuous Context Extraction"
---

Domain knowledge doesn't stay accurate on its own. Metric definitions, business rules, and table descriptions live across GitHub repositories, dbt YAML files, Looker models, and data catalogs. The moment any of those sources changes, your domain starts falling behind.\
\
Continuous Context Extraction (ACE) closes that gap. You can now connect your domain directly to the systems where your business logic lives. ACE runs extraction on a scheduled or on-demand basis, detects what changed since the last run, and surfaces only new signals for your review. This means your domain stays perpetually aligned with the source of truth without requiring you to manually export or upload a file.

## How it works

Continuous Context Extraction follows four stages from connection to a published domain update.

<Steps>
  <Step title="Connect a source" icon="plug">
    Navigate to your domain's Context Extractor page and open the **Connections** tab. A five-step wizard guides you through: choosing a source type, authenticating, scoping the extraction to a repository or branch, setting a schedule, and validating credentials. The connection appears in your list with a **PENDING** status.
  </Step>
  <Step title="Extract" icon="wand-magic-sparkles">
    The first run performs a full scan of the connected source. Subsequent runs are incremental, processing only changes since the last extraction. When a run completes, you receive an in-app notification with the count of items requiring attention and a direct link to the review screen.
  </Step>
  <Step title="Resolve conflicts and duplicates" icon="code-merge">
    ACE compares every incoming context object against what already exists in your domain. Conflicting entries surface side-by-side so you can accept either version, write a custom merged definition, or defer. After conflict resolution, an embedding-based similarity pass flags semantically equivalent entries as duplicate candidates — you can accept a merge, keep both, or defer. No automatic merges ever occur.
  </Step>
  <Step title="Publish" icon="check-double">
    Publishing a resolved extraction increments your domain version number and stores a full snapshot. You can view a diff between any two versions or restore any prior version from the version history panel. Rollback creates a new version entry — history is append-only.
  </Step>
</Steps>

<Note>
  Looker and dbt projects stored in GitHub are available immediately via the GitHub connection. Atlan, Glean, and Collibra are in active development and will appear in the connection wizard as they go live.
</Note>

## Connection statuses

| Status | Meaning |
| :-- | :-- |
| **PENDING** | Connection created, waiting for the first extraction to begin. |
| **EXTRACTING** | Extraction is in progress. |
| **AWAITING REVIEW** | Extraction complete. Items are ready for your review. |
| **ACTIVE** | All items reviewed and published. |
| **ERROR** | Extraction failed. A manual re-run option is available. |

## Target users

Continuous Context Extraction is available to:

- **Domain Admins**: Connect sources, configure schedules, review extractions, and publish changes.
- **Explorers and Analysts**: Benefit through improved answer accuracy and richer "Context Used" explanations on query responses. No action required.

## FAQs

<AccordionGroup>
  <Accordion title="Does anything get published to my domain automatically?">
    No. Every extraction goes through a human-in-the-loop review. There are no auto-merges in the current release.
  </Accordion>

  <Accordion title="What if I close my browser mid-extraction?">
    Nothing is interrupted. Extraction jobs run as persistent background processes. You can return at any time and the job will still be running or waiting for review.
  </Accordion>

  <Accordion title="What if I publish something and realize it was wrong?">
    Open the version history panel, find the prior version, and restore it with a single confirmation step. History is append-only — nothing is ever permanently deleted.
  </Accordion>

  <Accordion title="What's the difference between a conflict and a duplicate?">
    A conflict is when the same named term has different definitions across sources, detected by name match. A duplicate is when two differently-named entries are semantically equivalent, detected by embedding similarity. Both require your review before a new version is published.
  </Accordion>

  <Accordion title="How is this different from the original file-upload Context Extractor?">
    The file-upload extractor is a one-time, manual flow designed for bootstrapping. Continuous Context Extraction connects to live source systems, extracts on a schedule, and keeps the domain continuously aligned — no manual export or file handling required.
  </Accordion>
</AccordionGroup>
