Agents are automated workflows that monitor your data, run analyses, and deliver results on a schedule — without requiring manual queries. Instead of checking a dashboard every morning, you configure an agent once and it runs on its own, notifying you only when it matters.

WisdomAI offers two ways to build an agent. Choose the one that fits how you work.

## Prompt Mode

Prompt Mode is the fastest way to get started. You describe what you want the agent to do in plain language, and WisdomAI builds it for you. This mode works well for straightforward monitoring tasks — tracking a metric, flagging a threshold, sending a weekly summary.

## Visual Mode

Visual Mode gives you a drag-and-drop canvas where you connect nodes to define exactly how your agent fetches, analyzes, and delivers data. Each node has a focused responsibility, and outputs flow from one node to the next. This mode is suited to multi-step workflows where branching logic, conditional delivery, or multiple data sources are involved.

Both modes share the same components — triggers, scheduling, a variable store, the Run Visualiser, and a lifecycle that takes your agent from draft to active — and both can use the AI Builder to generate or modify a workflow from a natural-language prompt at any point.

## When to use an agent

Agents are most useful when you need consistent, recurring analysis without manual effort. The scenarios below show two common patterns.

### Monitor for problems

Use an agent to watch a metric and alert you only when something goes wrong.

- **Trigger:** Sales fall below $100,000 in the last day.
- **Work:** Send a list of lost deals from the last day, including region, owner, and total value.

This setup means a sales manager is alerted only when performance dips — no daily report to check, no threshold to remember.

### Track significant changes

Use an agent to detect meaningful fluctuations in either direction.

- **Trigger:** Monthly revenue moves outside a 4% range compared to last month.
- **Work:** Break down revenue by product category and compare it to the previous month.

This keeps the finance team aware of unusual volatility so they can act quickly — whether replicating a spike or addressing a drop.
