---
title: "Build an Agent - Visual Mode"
---

This guide walks through building an agent using Visual Mode in WisdomAI, from selecting a trigger to activating your workflow for delivery. If you prefer to describe your agent in plain language and have WisdomAI generate the structure for you, see [Create an agent - Prompt Mode](/advanced-features/agents/tutorial-create-agent).

<Note>
  The Agents feature is currently in beta.
</Note>

## Before you begin

Make sure you have the following:

- An active WisdomAI account with access to the **Agents** tab.
- A domain connected to the data you want the agent to work with.
- A delivery channel ready to configure: email, Slack, or webhook.

## Step 1: Open the Agent Builder

Visual Mode is the default view when you open the Agent Builder. The node canvas opens automatically and is ready for you to build.

1. Navigate to the **Agents** tab in the left sidebar and click **Create Agent**.
   <Frame>
     ![Create Agent Builder](/images/create-agent-builder.png)
   </Frame>
2. Select the **domain** your agent will run in.
   <Frame>
     ![Select Domain](/images/select-domain.png)
   </Frame>
3. (Optional) Edit your agent's name by clicking on the pencil icon. Add it in the **Agent Settings** modal and click **Save**.
   <Frame>
     ![Setup Agent Name](/images/setup-agent-name.png)
   </Frame>

### Use the WisdomAI Builder to generate your workflow

Rather than placing nodes manually, you can describe your agent in plain language and let the **WisdomAI Builder** generate the structure for you. The WisdomAI Builder is an AI-powered chat panel built into the canvas. Type what you want your agent to do in the **Describe your agent** bar at the bottom of the screen. WisdomAI Builder reads your connected domain and data sources, constructs the complete node structure, and renders it on the canvas for you to review, adjust, and publish.

<video src="/videos/builder-ia.mp4" controls />

You can use WisdomAI Builder at any point, such as to create an agent from scratch or to modify an existing workflow. For example: _add a conditional branch that skips delivery if revenue is above target._

If you prefer to place and configure nodes manually, continue to Step 2.

## Step 2: Select a trigger

Every agent starts with a trigger node, which defines when the agent runs.

1. Click the trigger node on the canvas. The **Select trigger** panel opens on the right.
2. Choose a trigger type.

| Trigger | When to use |
| --- | --- |
| **Manual trigger** | Starts when you run the agent manually. Use this during building and testing. |
| **Schedule** | Starts on a recurring schedule you define. Use this for production agents. |

3. If you want the agent to act only when a condition is met, enter it in the **Trigger Criteria** field. Leave it empty to run unconditionally every time the trigger fires.

<Tip>
  Start with a single condition. Once you confirm it fires correctly, add more. Debugging a multi-condition trigger from scratch is significantly harder than layering conditions one at a time.
</Tip>

4. Click **Done**.

## Step 3: Add and connect nodes

Nodes are the building blocks of a Visual Mode agent. Each node has a single, focused responsibility. You connect them using edges — click the **\+** button on any node's output to add the next node in the flow.

The table below shows the available node types and what each one does.

| Category | Node | What it does |
| --- | --- | --- |
| Data | **Data** | Fetches a dataframe from your connected data lake via SQL or Python execution. |
| Data | **Web Search** | Searches the web and brings results into the workflow as context. |
| Data | **MCP** | Calls an MCP tool to act across connected surfaces. See [Connect an MCP server](/integrations/mcp) for setup. |
| Reasoning | **Analysis** | Operates on a dataframe to perform filtering, aggregation, ordering, or scoring. |
| Reasoning | **Summary** | Takes a dataframe or context as input and produces a textual summary. |
| Control | **If/Else** | Evaluates a condition and routes the workflow down the matching branch. Inactive branches are skipped entirely. |
| Artifact | **Artifact Builder** | Ingests dataframes and context from the workflow run to produce a formatted report for delivery. |
| Delivery | **Email** | Sends the output to one or more email addresses. |
| Delivery | **Slack** | Posts the output to a Slack channel. |

<Note>
  Every agent requires at least one Artifact Builder node and one delivery node before it can be published.
</Note>

### Write effective node instructions

Each node's instruction tells it what to do with its input. A well-written instruction produces consistent, useful output — a vague one produces inconsistent results.

Keep these rules in mind when writing instructions:

- Write each instruction so it makes sense on its own, without referencing other nodes by name. Describe the input data the node should expect: "Given the list of assignees from the incoming issues, fetch their last 10 resolved tickets."
- Describe the outcome you want, not the procedure for producing it. Tell the node what to produce, not how.
- Assign one responsibility per node. If an instruction covers two unrelated concerns, split it into two nodes.
- For If/Else conditions, use concrete values — "the total count of items with status 'error' is greater than 10" rather than "there are many errors."

### Route data correctly through If/Else nodes

If data is needed on both sides of an If/Else node, connect it through the node — do not wire directly around it. Bypassing the node causes both branches to activate regardless of which condition is met.

### Connect data nodes directly to the Artifact Builder

If the artifact needs tables or charts, connect your Data or Analysis nodes directly to the Artifact Builder. Routing everything through a Summary node first strips the structured data the Artifact Builder needs to render tables and visualizations correctly.

## Step 4: Configure the Artifact Builder

The Artifact Builder formats your workflow's output into a finished report for delivery.

1. Click the **Artifact Builder** node on the canvas to open its settings.
2. Write an instruction that covers two things separately: how to organize the data into sections, lists, and tables; and how to format the final output — headings, number formats, and layout.

Be explicit about formatting. The Artifact Builder does not infer structure on its own. Include details like:

- Section headings: "Section 1: Overview", "Section 2: Details"
- Table columns: "Include columns: Deal Name, Stage, Days Stuck, Owner"
- Number formats: "Show currency as \$X,XXX" and "Show percentages as X.X%"
- Conditional sections: "Only include the Incidents section if incidents occurred"

3. Click **Done**.

## Step 5: Configure delivery

The delivery node sends your artifact to its destination.

1. Click **\+** after the Artifact Builder node and select **Email**, **Slack**, or **Webhook**.
2. Enter the destination — an email address, Slack channel name, or webhook URL.
3. Click **Done**.

<Note>
  To deliver to Slack, a workspace administrator must first connect your Slack workspace to WisdomAI. See [Connect to Slack](/integrations/connect-to-slack).
</Note>

## Step 6: Set a schedule

If you selected **Schedule** as your trigger, define when the agent runs.

1. Click the trigger node to reopen its settings.
2. Set a start date and time.
3. Select a frequency.

Match the frequency to how often your data changes and how quickly you need to act on it.

| Frequency | When to use |
| --- | --- |
| Hourly | Real-time monitoring only — production errors, critical SLA breaches |
| Daily | Most agents — recommended default |
| Weekly | Trend analysis, team performance summaries |
| Monthly | High-level business reviews, budget tracking |

## Step 7: Preview and test

Before publishing, run the agent manually to verify the output is correct.

1. Click **Preview** in the top right corner.
2. Watch the canvas as each node executes. A green check confirms the node succeeded. A red X indicates a failure — click the node to see the error.
3. After the run completes, open the **Run Visualiser** to review the data generated at each node, the analysis performed, and the SQL or Python used.
4. Adjust node instructions, trigger criteria, or formatting as needed and preview again.

### Common fixes

| Problem | Fix |
| --- | --- |
| Output is too broad | Add filtering criteria and exclusions to the instruction |
| Data is missing | Name the specific fields or columns you need |
| Formatting is inconsistent | Add explicit section headings, column names, and number formats to the instruction |
| Trigger fires too often | Tighten the threshold value |
| Trigger never fires | Loosen the threshold value |
| Node fails with "unsupported input" | Check that a Data node is connected and feeding into the failing node |

## Step 8: Publish and activate

Once the preview output looks correct, make the agent live.

1. Click **Publish**.
2. Click **Activate**.

The agent will run on the schedule you configured and deliver output to the channel you set. You can monitor past runs, edit the agent, or adjust the schedule at any time from the **Agents** tab.
